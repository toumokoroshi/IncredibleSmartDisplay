実装方針(適用時)

  • 切替方式: 時間帯で自動切替。dashboard.config.ts の theme を { mode: "auto", darkStartHour: 19, darkEndHour: 6 } に拡張(現在の未使用フィールド mode: "dark" を置き換え)。
  • 適用方法: 既存の useNow 系フックで現在時刻から light | dark を判定し、 <html> に data-theme 属性を設定。index.css に [data-theme="dark"] の変数上書きブロックを追加。
  • ハードコード色の変数化: index.css 内の直書き #0f172a / #64748b 等と、 各Widget内のTailwind色クラス(約220箇所)は、既存の .widget-card 再マッピングの仕組みに寄せて 変数参照に段階的に置き換え。レイアウト・サイズは不変更。
  • ステータス色: Trafficの運行状態バッジ(緑/黄/赤系)はダークでは明度を上げた変種 (#6ee7b7 / #fbd38d 系)+低透過の背景にし、暗背景でもコントラスト比 4.5:1 以上を維持。
  • 検証: 1524×1016 のタブレットビューポートで両テーマの表示確認、 npm run build / test:run / lint、切替時刻境界のユニットテスト追加。
